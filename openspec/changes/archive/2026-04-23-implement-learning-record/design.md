## Context

根据 constitution.md 的域模型定义，LearningRecord（学习记录）是一个核心实体，用于存储用户对知识点的学习反馈。关键特性包括：

- **学习记录不可更新**：学习记录是历史数据，一旦创建不可修改或删除（如需修正，创建新记录）
- **关联关系**：每条记录关联一个用户（User）和一个知识点（KnowledgePoint）
- **核心字段**：学习时间（startTime）、学习时长（durationMinutes）、掌握程度（masteryLevel: A/B/C/D/E）、备注（notes）

前置变更 `implement-knowledge-learning-ui` 已完成学习界面的搭建，包含：
- 左侧知识树（树状/思维导图视图）
- 中间内容区（Markdown 预览）
- 右侧 AI 侧栏
- 下方反馈区（占位，待本变更实现）

## Goals / Non-Goals

**Goals:**
- 实现学习记录的创建和查询 API
- 开发学习反馈面板（下方反馈区）
- 实现掌握程度五级评分组件（A/B/C/D/E）
- 开发学习计时器（自动记录学习时长）
- 实现历史记录列表展示

**Non-Goals:**
- 不实现学习记录的更新和删除（根据 constitution.md，学习记录不可修改）
- 不实现学习数据的统计分析（后续变更处理）
- 不实现学习提醒功能
- 不实现学习目标设定

## Decisions

### 1. 学习记录仅可创建（不可更新/删除）
**理由：**
- 学习记录是历史事实数据，不应被修改以保证数据真实性
- 如需修正，创建新记录覆盖即可
- 简化 API 设计，无需 PATCH/DELETE 端点

### 2. 计时器采用客户端计时 + 服务端记录模式
**理由：**
- 客户端计时提供实时反馈，用户体验更好
- 服务端记录实际提交时的 durationMinutes，作为权威数据
- 防止客户端篡改，以服务端数据为准

### 3. 掌握程度使用 A/B/C/D/E 五级评分
**理由：**
- 符合 constitution.md 定义的枚举值
- 对应不同颜色：A(优秀/绿)、B(良好/浅绿)、C(一般/黄)、D(较差/橙)、E(很差/红)
- 五级评分足够区分掌握程度，不过于复杂

### 4. 反馈面板采用折叠式设计
**理由：**
- 学习页面空间有限，折叠设计节省空间
- 用户学习完成后展开填写反馈
- 默认展开显示历史记录列表，方便查看

### 5. 学习记录列表按时间倒序排列
**理由：**
- 最新学习记录对用户最有价值
- 便于追踪最近的学习活动
- 与常规日志展示模式一致

### 6. NestJS 模块化设计
**理由：**
- 遵循 constitution.md 的 NestJS 架构原则
- LearningModule 包含 Controller + Service + DTO
- 便于后续扩展（如学习分析功能）

## Risks / Trade-offs

| 风险 | 缓解措施 |
|-----|---------|
| 学习记录数据量增长快 | 后续变更添加分页查询和数据归档策略 |
| 客户端计时可能不准确 | 以服务端记录的 durationMinutes 为准，客户端仅作参考 |
| 用户忘记提交反馈 | 考虑后续添加页面离开提醒或自动保存草稿功能 |
| 掌握程度评分主观性强 | 后续添加学习测试功能，用客观数据辅助评估 |
