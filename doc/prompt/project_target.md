## 项目目标
**角色:** 你是一个经验丰富的系统架构师且精通编程开发，具备有快速构建稳定系统的能力；
**目的:** 请帮我创建一个web系统，名称为"数学通"，展示指定教材的大纲知识点，进行知识点的学习，并记录学习情况以进行分析。

## **技术栈要求**
1.前端技术要求使用React+TypeScript+Vite；
2.后端技术要求NestJS+Prisma+PostgreSql;
3.使用文件存储知识点框架(*.xlsx)和知识点详情(*.md)；
4.使用PostgreSql保存学生角色用户对知识点的反馈；
5.支持基于LaTex的数学公式展示，mermaid图表展示，echarts分析图表展示；
6.使用标准的RESTful API设计规范，实现前台和后台的严格分离，独立部署和启动；
7.采用pnpm作为包管理工具，实现快速部署和依赖管理；

## **功能要求：**
### 知识库管理
1. 知识库通过在指定文件夹(/iksm/)文件方式存放，
2. 知识库内容展示分成左右两部分展示，左侧展示对应的文件夹下内容，右侧展示文件夹对应的知识库库列表；
3. 左侧文件夹内容可通过上传功能添加文件，也可通过删除功能删除自定文件；
2. 点击刷新按钮，可刷新右侧知识库列表，实现发现新教材（通过文件增加）和知识点架构和详细内容的变化（通过文件日期变化）；
3. 每本教材知识点文件有两个组成知识点框架和知识点详情，其文件名相同，后缀不同，如math01.xlsx/csv为知识点框架，math01.md为详细知识点，教材名称为math01；
4. 文件夹（/iksm/）下只能有类似math01.xlsx/csv和math01.md文件，其他后缀文件会被忽略，文件名以".,_,tmp"开头的文件也会被忽略；
5. csv文件可以看做是一种特殊的excel文件,与xlsx后缀的知识点架构文件功能相同，也同时支持。

### 知识点学习
1. 知识点展示界面通过灵活布局的多栏式界面来实现，类似TRAE的左侧文件浏览区，代码编辑区，右侧AI侧栏区，下方终端命令区,各区域可根据用户需求进行调整和隐藏；
2. 左侧区使用树状结构展示知识点框架，树状结构要使用比较清晰的图标和易用的展开方式,可呈现数图和思维导图两种方式；
3. 中间展示区，展示md文件的预览效果，支持预览mermaid图形，并可使用按钮选项展示图形或原始md文件；
4. 左侧知识点框架要联动中间展示区，中间展示区展示时可以选择只显示当前知识点和上下文知识点；
5. 右侧AI侧栏区，展示对当前知识点的示例、例题和讲解等扩展功能。其内容暂时为空，后期再扩展功能和内容来源；
6. 下侧展示对知识点的学习反馈，包括学习时间，学习时长，备注、掌握程度打分(A\B\C\D\E), 用户（即学生）在学习的过程中可填写上面的反馈，并保存历次的记录;
7. 可根据知识点反馈情况进行针对性性学习，输出对掌握情况不好知识点(C\D\E)，让学生持续学习，并可记录学习反馈；
8. 可根据知识点重要性级别进行重点学习，单独学习A类知识点，然后学习B类知识点，最后学习C类知识点，并可记录学习反馈；
9. 可随机输出教材的知识点进行随机学习，并可记录学习反馈

### 知识点分析
1. 分析指定用户（即学生）对知识点的学习次数，学习时长；
2. 分析指定用户（即学生）对知识点的掌握情况；

### 用户管理
1. 系统分为学生角色用户和教师角色用户；
2. 学生角色用户只能学习和查看知识点，并可以反馈自己对知识点的掌握情况和备注信息，可查看分析能力；
3. 教师角色用户可实现对知识库中教材和知识点的管理；
4. 教师角色内置管理员账号，可对系统进行全面管理，包括用户账号管理、权限设置、系统配置等；
5. 学生角色通过注册账号并填写个人信息（如姓名、学号、班级等）进行账号创建；
